Git实战

你是如何看到这个博客的

张蓝翔
9 min

# 1. 下载Git

进入官网，点击下载，选择电脑的版本，我这里选的是【64-bit Git for Windows Setup】。官网地址：https://git-scm.com/

安装过程中会有很多选项，除了安装地址，其他默认即可。

# 2. Git核心原理

本节的知识，本人是观看**@技术蛋老师**的视频了解的，想深入学习的观众可以关注他的频道。

在技术蛋老师的视频中，他举了一个例子：

  • 每个家族都有一本武林秘籍，家族里的每一个人都会抄写一份副本。
  • 当甲想出了新的招式时，会把动作记录下来。
  • 甲把动作练得差不多之后，会把新的招式起个名，记录到副本当中。
  • 甲会用副本和原版对比，并把新的招式添加到原版中。
  • 原版更新后，家族里的每一个人都会重新抄写新的副本。

而实际上，Git的工作原理是：

  • 每个项目的代码都会存储在一个远程仓库中，程序员会从这个远程仓库将这个项目clone到工作区。
  • 当程序员甲在本地修改了代码，就会把修改后的那部分代码add到暂存区。
  • 当甲觉得代码修改没什么问题后，会将代码commit到本地仓库中。
  • 甲可以将本地仓库的新内容push到远程仓库中，更新远程仓库的代码。
  • 远程仓库更新后，程序员都可以将新的代码pull到工作区。
    • pull会把工作区都更新了，如果工作区有未保存、未提交的部分会被清空。可以使用fetch先将新的代码下载到本地仓库，再通过diff和工作区进行对比，当没问题了就可以正式合并到工作区。

# 3. 体验Git语句

  1. 创建一个空文件夹。

  2. 操作前，最好先进行用户的配置，方便以后查看操作人是谁。

    git config --global user.name zlx
git config --global user.email "you@example.com"

  3. 对这个文件夹开启版本控制，也就是初始化git工作区。

    git init

    初始化后文件夹会多出一个.git文件夹；并且在Git Bash中会显示当前分支为master。

    新建一个文件123.txt，可以使用status语句查看当前工作区的状态。

    git status

  4. add。add语句负责将需要上传的文件从工作区移到暂存区。

    git add 123.txt

    add操作执行后，再次输入git status来查看状态，会说这里有个文件可以被commit。

  5. commit。输入commit命令以提交暂存区的修改。

    git commit

    输入以上命令后会进入vim编辑器。输入i（表示insert）可以插入内容（这个内容是对本次提交内容的描述）；输入完成后点击ESC以完成插入；输入【:wq】以退出vim编辑器。退出后就算是commit完成了。此时再用git status查看，可以看到nothing to commit。当然也可以不这么麻烦，通过-m快捷提交。

    git commit -m "内容"

    还可以将add和commit写到一起，快捷提交。

    git commit -a -m "内容"

    在commit了之后，可以通过rm指令回到add前的状态。

    git rm 123.txt // 删掉
git rm --cached 123.txt //保留但是不跟踪

  6. git ignore。创建.gitignore文件，在内部写出不需要提交的文件名。可以通过以下语句来创建。

    touch .gitignore

    创建456.txt，并且输入以上语句。先不在.gitignore文件文件中配置需要忽略的内容，输入git status，能够看到需要提交的文件是包括了456.txt的；而在.gitignore文件中输入了456.txt后，再输入git status，需要提交的文件里就不包括它了。

  7. 创建分支。什么时候需要分支？分支就是一条if线，当有某份文件还不知道需不需要的时候，就可以在主分支的基础上创建一个分支，最后再合并到主分支。创建分支语句如下。

    git branch 分支名

    通过上述命令创建分支后，不会马上进入分支。当然，也可以在上面的语句加入【-b】以直接进入分支。可以通过下述命令查看已有哪些分支。

    git branch

    切换分支则需要下面的命令：

    git checkout 分支名

    切换分支后，删除123.txt和456.txt两个文件。然后commit分支，提交成功后，切换回主分支。可以发现123.txt回来了，但456.txt没有，那是因为再.gitignore文件中输入了456.txt，git就不管他了，死活都不管。

    删除分支的语句如下：

    git branch -d 分支名

    删除分支之前，如果还没有合并分支到主分支中会提示。将其他分支合并到当前分支的语句如下：

    git merge 被合并的分支

  8. push。要想push，首先得先有远程仓库。在GitHub点击【new repository】创建仓库。输入各种信息之后，点击确定，这时候会出现很多东西，点下图这个按钮创建一个文件即可。

    然后就能创建文件了。这里我们还是创建一个123.txt。设置好文件名并且输入完文件内容后，点击commit changes，就相当于我们在本地输入了【git commit -ma "内容"】。

    提交完成后，就能看到远程仓库此时已经是有东西的了，如下图所示。

    如何下载呢？可以通过下载压缩包的方式，也可以通过git clone的方式。使用压缩包的方式是没有.gitignore的，所以选择git clone。复制网址，然后打开cmd，输入以下内容即可。

    git clone 网址

    当在本地id工作区修改了项目之后，可以通过以下语句查看当前工作区和哪些远程仓库有联系。

    git remote -v

    能查看到的结果有两条，其中远程仓库的名字为origin。

    可以通过push语句直接提交。第一次输入时，应该会弹出一个提示框，让你验证身份，本人采取的是使用网页验证。验证成功后，就能成功push了。事实上是要输入远程仓库的名字和要被上传的分支名的。

    git push origin master

  9. fetch、diff、pull。当在Github更新了远程仓库，需要重新获取时，输入以下语句，将远程仓库的内容下载到本地仓库，然后比较origin即远程仓库和main分支的内容。

    git fetch
git diff origin/main

    如果看diff的结果没什么问题，就可以运行pull语句来获取分支。

    git pull

  10. 暂存。有时候不得不从一个分支切换到另外的分支，但当前的分支还不能提交，就可以通过stash语句暂时保存。

    git stash

    当从别的分支切换回来的时候，可以重新应用暂存的内容。

    git stash apply

  11. 撤回提交。通过reset语句来实现撤销效果。

    git reset head~ --soft

    其中，head~表示当前提交的前1次提交。不加~则表示当前的提交，head~2则表示前2次提交。而--soft表示，只是撤销commit操作，不撤销add操作；如果没有--soft，只需要git add 文件即可。

# 4. 通过Git上传博客

首先要明白，把博客挂到git上面要做哪些操作：

  1. 打包博客。
  2. init、add、commit、push博客内容。

正式操作：

  1. 打包。npm run docs:build

  2. 进入文件夹。cd docs/.vuepress/dist

  3. git init

  4. git add -A

  5. git commit -m "这是描述"

  6. git push。照理来说是git push，但这时候还没告诉git要上传到哪、要上传到哪个分支，输入git push后，它会提示你要如何操作，如图所示。

    我们照着它做。首先是关联远程仓库。其中，origin是为远程仓库起的名字。后面的链接就是远程仓库的地址。

    git remote add origin https://github.com/kitouakari/kitouakari.github.io.git

    然后开始push操作。

    git push -u origin master

    上面的语句中，-u的作用是记录push到远端分支的默认值，相当于以下的语句。

    git push origin master
git branch --set-upstream master origin/master

大功告成，所有人都可以联网访问这个博客了。我个人博客的地址是https://kitouakari.github.io/
